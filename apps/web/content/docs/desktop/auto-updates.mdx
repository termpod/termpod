title: Auto-Updates
description: Desktop app auto-update mechanism via tauri-plugin-updater and relay proxy.
---

The desktop app self-updates using `tauri-plugin-updater`, with the relay server acting as a proxy for GitHub release artifacts.

## How It Works

1. Desktop checks `GET /updates/latest.json` on the relay
2. Relay proxies the request to GitHub releases API
3. Download URLs are rewritten to point through the relay (enabling private repo updates)
4. Desktop downloads the update via `GET /updates/download/:filename`
5. Update is verified with ECDSA signature
6. User is prompted to install

## Relay Proxy Endpoints

### `GET /updates/latest.json`

Returns the latest release metadata from GitHub. The relay fetches from the GitHub API using `GITHUB_TOKEN` and rewrites download URLs to go through its own proxy.

### `GET /updates/download/:filename`

Proxies the actual binary download from GitHub releases.

## Configuration

The relay needs a `GITHUB_TOKEN` with read access to the repository's releases:

```bash
wrangler secret put GITHUB_TOKEN
```

If `GITHUB_TOKEN` is not configured, the relay returns 204 (no update available).

## Signing

Updates are signed with a private key:

```bash
# Set in shell environment (NOT .env file)
export TAURI_SIGNING_PRIVATE_KEY="content-or-path-to-private-key"
export TAURI_SIGNING_PRIVATE_KEY_PASSWORD=""
```

The corresponding public key is embedded in `tauri.conf.json` under the `updater` config.

## Update Banner

The `UpdateBanner` component (`apps/desktop/src/components/UpdateBanner.tsx`) shows a notification bar when an update is available, with options to download and install.

## Hook

The `useUpdater` hook (`apps/desktop/src/hooks/useUpdater.ts`) manages the update lifecycle:

- Periodic update checks
- Download progress tracking
- Install triggering
