The TermPod desktop app is a macOS application built with **Tauri 2.0** (Rust backend + React frontend). It is the only component that runs an actual shell — spawning PTY sessions locally with full access to the filesystem, credentials, and environment.

## Tech Stack

- **Tauri 2.0**: Rust backend with WKWebView frontend
- **React 19**: UI framework
- **xterm.js 6.0**: Terminal rendering (with WebGL, fit, search, web-links, unicode-graphemes addons)
- **tauri-plugin-pty 0.2.1**: PTY spawning and management
- **TypeScript 5.8**: Type-safe frontend code

## Key Responsibilities

1. **PTY Management**: Spawns shell sessions, manages resize, tracks working directory
2. **Terminal Rendering**: xterm.js with WebGL acceleration, search, clickable links
3. **Multi-Session Tabs**: Multiple concurrent terminal sessions
4. **Transport Management**: Connects via Bonjour, WebRTC, and relay
5. **E2E Encryption**: Encrypts all outgoing terminal data
6. **Auto-Updates**: Self-updates via relay-proxied GitHub releases

## Configuration

All user settings are stored as JSON files in `~/.termpod/`:

| File | Purpose |
| --- | --- |
| `config.json` | Appearance, terminal, and behavior settings |
| `keybindings.json` | Custom keyboard shortcut overrides |
| `workflows.json` | Saved command workflows |
| `themes/*.json` | Custom terminal color themes |

Files are watched for changes — edit in any text editor and the app updates live, no restart needed. See [Customization](/docs/desktop/customization) for the full reference.

## Important Implementation Notes

- **Tab panels use `position:absolute + visibility:hidden`** — not `display:none`, which breaks xterm.js WebGL rendering in zero-dimension containers
- **Terminal callbacks stored in refs** — avoids xterm recreation on re-render
- **Menu handler uses ref pattern** — avoids stale closures
- **Relay state propagated via ref-based `onRelayChange`** — prevents render loops
- **PTY onData returns `Uint8Array`** — not string (tauri-plugin-pty behavior)
