The TermPod iOS app is a native **SwiftUI** application that serves as a viewer and input device for terminal sessions running on the desktop. It does not run a shell itself.

## Tech Stack

- **SwiftUI**: UI framework
- **SwiftTerm 1.11.2**: Terminal rendering via SPM
- **LiveKit WebRTC**: WebRTC P2P via `livekit/webrtc-xcframework`
- **CryptoKit**: E2E encryption (ECDH + AES-256-GCM)
- **SentrySwiftUI**: Error tracking

## Project Configuration

- **Bundle ID**: `dev.termpod.mobile`
- **Deployment target**: iOS 17.0
- **Swift version**: 5.9
- **Project generation**: XcodeGen from `project.yml`
- **Config**: `Config.xcconfig` generated from `.env` via `generate-config.sh`

```bash
# Generate Xcode project
pnpm ios:generate

# Open in Xcode
open apps/ios/TermPod.xcodeproj
```

**Important**: Edit `project.yml` for project changes, not `.xcodeproj` directly.

## Key Classes

### `AppState`

Global `@Observable` state machine that manages:
- Authentication state
- Device list and connection status
- Active session tracking
- Transport status

### `DeviceTransportManager`

Orchestrates three transport layers concurrently:
- Spawns Local, WebRTC, and Relay transports
- Each transport independently connects and reports status
- Selects best available transport for sending
- Receives data from all active transports

### `ConnectionManager`

High-level connection logic:
- Manages transport lifecycle
- Handles session subscriptions
- Exponential backoff reconnection
- Transport fallback

### `CryptoService`

E2E encryption using CryptoKit:
- ECDH P-256 key exchange
- HKDF-SHA256 key derivation
- AES-256-GCM encryption/decryption
- Must stay in sync with desktop implementation

## Info.plist Requirements

The iOS app requires these entitlements for local networking:

```
NSLocalNetworkUsageDescription — Bonjour discovery
NSBonjourServices — _termpod._tcp
```
